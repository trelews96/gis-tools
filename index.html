Launching the Tool Panel

1

Open your ArcGIS web map

Navigate to your Map Viewer or Experience Builder app and wait for the map to fully load.

2

Click the 🔧 GIS Tools bookmark

Click the bookmark in your bookmarks bar. The dark GIS Tools panel will appear in the top-left corner of the page.

3

Click any tool to load it

Tools load on demand from GitHub when you click them. The first load may take a moment — watch the status bar at the bottom of the launcher for confirmation before using the tool.

4

Close tools when finished

Use the ✕ on individual tool panels to close them, or click ⬛ Close All Tools at the bottom of the launcher to reset everything at once.

Panel Controls

✋

Drag to Reposition

Click and drag the panel header bar to move the launcher anywhere on screen so it doesn't block your map.

▾

Collapse / Expand

Click the ▾ button in the header to collapse the launcher to just the title bar, freeing up screen space while keeping it accessible.

✕

Close the Launcher

The red ✕ button closes the launcher and all open tools. Click the bookmark again at any time to relaunch it.

📂

Tool Groups

Click any group header (Geometry, Packages, etc.) to collapse or expand that category to keep the panel tidy.

General Tips & Gotchas

⚠️ Tools modify live map data. Always confirm you are on the correct web map and work order before making edits.
💡 If a tool doesn't open or behaves unexpectedly, try refreshing the page and clicking the bookmark again — this fully reinitializes the host.
💡 The launcher resets on page refresh. This is by design — each session starts clean.
⚠️ On slow connections, tools may take a few seconds to load the first time. Wait for the ✅ confirmation in the status bar before interacting with the tool panel.

GIS Edit Tools
Move features, edit vertices, split lines, and copy features on the map
▾
Description
A lightweight editing toolbar that attaches to your ArcGIS web map without replacing the native editor. Provides four focused editing modes in a single draggable panel: moving features and vertices with automatic snapping, locking edits to a single feature when features overlap, splitting lines at a point location, and placing copies of existing features at clicked locations.
Move & Vertex Tools
  1. 1Click Enable Tool to activate editing mode.
  2. 2Choose Point Mode to move point features, or Line Mode to move individual line vertices. Connected line endpoints follow automatically in Point Mode.
  3. 3Click the feature or vertex to select it, then click the destination to move it. The destination snaps to the nearest point or vertex within tolerance.
  4. 4In Line Mode, use Add Vertex / Delete Vertex to insert or remove vertices on any visible line.
  5. 5Toggle Show Vertices to display all line vertices as a colored overlay. Click Refresh to update after panning.
Single Feature Lock
  1. 1Click Pick Feature, then click a feature on the map to lock all edits to that feature only.
  2. 2Perform your edits — only the locked feature is affected, even if others overlap at the same location.
  3. 3Click Release Lock when done to return to normal editing mode.
Cut & Split Lines
  1. 1Click Enable Cut Mode, then click a point feature on the map.
  2. 2A confirmation menu lists all intersecting lines found within a 15 ft buffer. Hover over each to highlight it on the map.
  3. 3Uncheck any lines you do not want to cut, then confirm. Each selected line is split into two new segments at the point location.
  4. 4Click Undo Cut to reverse the most recent cut batch if needed.
Click & Copy
  1. 1Click Enable Copy Mode, then click a feature on the map to use it as a template.
  2. 2If multiple features overlap at the click location, a picker menu lets you choose the correct template.
  3. 3Each subsequent click places a copy at that location with snapping and a live crosshair preview.
  4. 4Press ESC or click Clear Template to pick a new template without leaving copy mode.
Tips & Gotchas
⚠️ Use Single Feature Lock whenever features overlap — without it, coincident vertices on multiple lines may all move together.
💡 The Undo Cut stack only covers the current session and cannot be used after a page refresh.
💡 Snapping is automatic — destinations will snap to the nearest point feature or line vertex within the configured tolerance.

Path Editor
Guided sequential attribute editing across multiple map features
▾
Description
Rather than clicking features one by one on the map, Path Editor lets you define a selection, configure which layers and fields to edit, and step through each feature in a structured queue. Supports single click, drawn path, and polygon selection modes. Includes a visual filter builder, bulk edit mode, drag-and-drop attachment uploads, per-layer CSV export, and a session report for audit trails.
How to Use
  1. 1Choose a selection mode: Single Click for spot edits, Line Path to select features along a corridor in sequential order, or Polygon to select all features within a drawn area.
  2. 2Click or draw on the map. Discovered layers appear as collapsible cards showing feature counts.
  3. 3Enable each layer you want to edit. Set each to Edit Fields or View Only, select which fields to include, and configure processing order.
  4. 4Optionally use the Filter Builder to narrow each layer's feature set using plain-English conditions. Click Test Filter to preview the match count before proceeding.
  5. 5Click Review & Start to enter editing mode. Features are presented one at a time with a progress bar at the top.
  6. 6Fill in fields, attach files if needed, then click Submit & Next. Use Skip to defer a feature or Previous to go back.
Tips & Gotchas
💡 Save your layer/field configuration after setup — repeat workflows can be reloaded instantly with the Load button.
💡 Apply Previous Values fills the current form with the last submitted feature's values — great for repetitive edits along a run.
💡 Download a CSV from any layer card before editing to review the exact data you're about to change in Excel first.
⚠️ Bulk Edit applies the same values to ALL selected features at once — verify your filter is correct before using it.
💡 Use Line Path mode when editing assets along a route (e.g., spans along a road) — features are queued in the order they appear along the drawn path.

UG Package
Place preconfigured vaults and auto-draw conduit and fiber between them
▾
Description
Speeds up underground package data entry by creating preconfigured vault features at each clicked map location, then automatically drawing conduit and fiber connections between each created vault in sequence. Eliminates manual feature creation and attribute entry for standard vault-to-vault runs.
How to Use
  1. 1Launch the UG Package tool and configure vault type and conduit/fiber settings as prompted.
  2. 2Click each vault location on the map in the order you want them connected. A vault feature is created at each click.
  3. 3When finished placing vaults, confirm to have the tool automatically draw conduit and fiber between each vault in the order they were placed.
Tips & Gotchas
⚠️ Ensure you are on the correct web map with the appropriate underground layers loaded and visible before starting.
💡 Plan your vault locations before clicking — connections are drawn in placement order and cannot be reordered without re-running.

QC Review
Streamlined pass/fail review workflow for completed features
▾
Description
Quickly identifies all features in a completed status that need QC review. Presents each feature one at a time — zooming to it and opening its popup for photo review — then provides a simple Pass / Fail / Missing Photo decision interface. Passing a feature marks it QC Complete; failing a feature creates a gig point and logs the selected issue types and notes.
How to Use
  1. 1Optionally filter by Work Order and/or Installation Date Range to scope your review queue.
  2. 2Click Query Features. The progress counter shows the total number of features found.
  3. 3The tool zooms to each feature and opens its popup. Review the photos and feature attributes.
  4. 4Select Pass, Fail, or Missing Photo. For failures, check the applicable issue type(s) and add optional notes.
  5. 5Click Submit QC → to record the decision and advance to the next feature. Use Skip to defer or Previous to go back.
Tips & Gotchas
💡 Use the date range filter to limit your queue to a specific installation window rather than reviewing an entire work order at once.
⚠️ Gig points created on failed features are written to the map immediately — failures cannot be undone through this tool.
💡 Sort by Newest to Oldest to prioritize the most recently installed features first.

Attachment Manager
Bulk upload or download photos for any layer, with optional timestamp watermarking
▾
Description
Handles mass photo management for map layers. Upload multiple photos to features at once, or download all attachments from a layer in bulk. Includes an optional watermark that stamps photos with a timestamp at upload time — useful for documentation and audit trail requirements.
How to Use
  1. 1Select the target layer from the dropdown.
  2. 2Choose Upload or Download mode.
  3. 3If uploading, enable the Timestamp Watermark option before selecting files if timestamping is required.
  4. 4Select or drag-and-drop your files and execute the operation.
Tips & Gotchas
⚠️ Enable the watermark option before selecting files — it cannot be applied retroactively after files are chosen.
💡 Bulk downloads are saved to your default browser Downloads folder. Check there after a large export.

Work Order Overview
Active work order dashboard with progress, run rates, and crew performance
▾
Description
Displays all active work orders as individual cards, each showing progress bars for Construction, Billing, and Invoice milestones alongside run rate estimations and estimated completion dates. Clicking any card opens a detailed view with a layer-by-layer production breakdown across all categories, invoice opportunity callouts, and a ranked crew performance table. Clicking any cell in the breakdown table filters the live map to that specific layer and category combination.
How to Use
  1. 1Optionally filter by Purchase Order and select which layers to include in the analysis.
  2. 2Click Load Overview to generate the work order cards.
  3. 3Use the sort dropdown to order cards by completion %, age, or other criteria.
  4. 4Click any work order card to open its Detailed Analysis view.
  5. 5In the detailed view, use the date range filter buttons to adjust the analysis window, and click any table cell to filter the map to that layer + category.
Tips & Gotchas
💡 Sort by Completion % (least first) to immediately surface the most behind-schedule work orders.
💡 The Invoice Opportunity callout in the detailed view shows units ready for daily submission — check this before submitting dailies.
💡 Crew performance ranks by constructed units and daily rate. Use this to identify high and low performers on a given work order.
⚠️ Run rates and estimated completion dates are based on pace within the selected date range — adjust the filter to see different projections.

Daily Tracking
Zoom to a daily, filter the map, and verify recorded quantities match actual feature quantities
▾
Description
Enter a daily number to immediately zoom to that daily's location, filter the map to show only features associated with that daily, and display all CWI labor codes tied to it. Then compares the quantity recorded in each feature's daily tracking table against the actual measured quantity of the feature itself, flagging any mismatches that need to be resolved before the daily can be billed.
How to Use
  1. 1Enter the daily number in the input field and click Run Analysis.
  2. 2The map zooms to the daily's extent and filters to matching features. All associated CWI labor codes are shown.
  3. 3Review the quantity comparison table. Any row where the daily tracking table quantity doesn't match the feature quantity is flagged as a mismatch.
  4. 4Click Zoom next to a flagged feature to navigate to it on the map and investigate.
  5. 5Correct mismatched quantities, then re-run the analysis to confirm everything is resolved before submitting for billing.
Tips & Gotchas
⚠️ Quantity mismatches will cause billing issues — always resolve all flagged features before submitting a daily.
💡 The map filter applied by this tool may persist after closing the tool. Reset map filters manually if the map still appears filtered afterward.

Parent/Child Reconciliation
Detect quantity mismatches between coincident fiber and strand/conduit for billing compliance
▾
Description
Finds locations where fiber is coincident with strand or conduit and compares the daily tracking table quantities between the paired features. Mismatches are flagged because composite billing codes require all payable code quantities to match exactly — a single mismatch will prevent the composite code from being billed out. The tool produces a sortable list of each problem location with fiber ID, labor codes, quantities, the signed difference, and direct zoom buttons.
How to Use
  1. 1Launch the tool and click Run Analysis. The tool scans for fiber features coincident with strand or conduit.
  2. 2Review the mismatch list — each row shows the Fiber ID, both labor codes, their quantities, and the difference.
  3. 3Click the UG or Fiber zoom button on any row to jump directly to that feature on the map.
  4. 4Correct the quantities in the daily tracking tables, then re-run to confirm all mismatches are resolved.
Tips & Gotchas
⚠️ Unresolved mismatches block billing for the composite code — run this check before submitting any daily that includes both fiber and strand/conduit.
💡 A positive Diff value means the UG quantity is higher; negative means the fiber quantity is higher.
💡 Use Export CSV to download the full mismatch list and share with the billing team for tracking.

Slack Loop Analysis
Find slack loop features that are missing sequential numbers
▾
Description
Scans slack loop features within a selected work order and identifies any that are missing sequential numbers. Generates a detailed list showing location type, OID, fiber count, minimum distance, and issue type so editors can quickly navigate to each problem location and assign the missing sequentials.
How to Use
  1. 1Select the Work Order ID from the dropdown.
  2. 2Set the Fiber Detection Tolerance (meters) — how close a fiber cable must be to a location to be counted.
  3. 3Set the Slack Loop Tolerance (meters) — the minimum distance a loop must span to be considered valid.
  4. 4Click Run Analysis. The summary shows totals for locations, fiber cables, slack loops, and issues found.
  5. 5Review the Missing Slack Loops table and click Zoom next to each entry to navigate to it and add the missing sequential.
Tips & Gotchas
💡 Start with the default tolerances and adjust if results seem over- or under-inclusive for your specific dataset density.
💡 Use Export CSV to share the missing loop list with editors or use it as a completion checklist.
⚠️ Make sure the full work order area is visible on the map before running — the analysis respects the current map extent.
